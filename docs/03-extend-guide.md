## 3. 拡張レシピ（例: カウントダウン上限を10秒→30秒にする）

1. 検索: `countSec` で全ヒットを確認（定義・clamp・UI表示の3系統が出る）
2. clampの `0,10` を `0,30` へ（保存側・復元側の両方）
3. HTML側の入力UI（max属性やボタン）を合わせる
4. 動的テストの境界値を書き換えて実行: `clamp(31,0,30) === 30` になること
5. 変更履歴に1行追記し、テスト全PASSを確認してから刻印

ポイント: 「定義・clamp・UI・テスト・履歴」の5点セットで1つの変更です。どれか欠けると、復元時に古い上限へ丸められる等の不可解なバグになります。

## 4. 動的テストの回し方（要約）

詳細は別冊「動的テスト入門」参照。最小構成は次のとおり。

```bash
npm init -y
npm i -D jsdom
node --test        # test/ フォルダの *.test.js を自動実行
```

- レベル1（ユニット）: extract.js で関数を切り出し、clamp／hexLum／contrastRatio等の境界値を検証（15本・検証済み）
- レベル2（スモーク）: jsdomでHTMLを丸ごと起動。起動確認・バージョン表示・通常版でAI相談が「使えません」表示になること・localStorage保存（v:12／cpm:300／size:48）を検証（5本・検証済み）
- ハマり3点: `after(() => dom.window.close())` を忘れるとテストが終わらない／内部関数は`button.click()`で駆動する／jsdomは`http://localhost`のURL指定でlocalStorageが使える

## 5. やってはいけないこと

- ES2020以降の構文（`?.` `??` 等）の持ち込み: 本ツールはES5互換方針
- 外部CDN・外部scriptの追加: 単一ファイル・オフライン完結の設計原則に反する
- APIキーのコード埋め込み: 認証は実行環境に委譲する設計（1899行のcallAI参照）
- テストを通さない刻印: PROは「全テストPASSの証」
