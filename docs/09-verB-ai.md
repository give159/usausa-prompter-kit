## 1. 前提：2つの動作モード

このツールは単一HTMLファイルで、**2つの環境**で動作します。

| 環境 | 判定方法 | AI相談 | その他の機能（読み上げ・保存・書き出し等） |
|---|---|---|---|
| **アーティファクト版**（Claude上） | `isArtifactEnv()` がホスト名を判定 | ✅ 使える | ✅ 使える |
| **通常版**（ローカルファイル直開き） | 同上 | ❌ 通信エラーになる | ✅ オフラインで使える |

判定ロジックは以下の通りです（コードそのまま）。

```javascript
function isArtifactEnv(){
  try{
    if(location.protocol==="file:")return false;
    const h=location.hostname||"";
    if(!h||h==="localhost"||h==="127.0.0.1")return false;
    return /claude\.site|claudeusercontent|anthropic/i.test(h);
  }catch(e){return false;}
}
```

`file://` で開いた場合や `localhost` は「通常版」と判定され、UI上部の相談バーに警告表示（`ai-bar warn`）が出ます。ホスト名に `claude.site` / `claudeusercontent` / `anthropic` を含む場合のみ「アーティファクト版」として案内されます。

**重要**: コード内にAPIキーは一切含まれていません。アーティファクト版でAI相談が動くのは、Claudeのアーティファクト実行環境が `https://api.anthropic.com/v1/messages` への呼び出しを認証込みで仲介しているためです（本ツール自体が鍵を保持・送信しているわけではありません）。

## 2. AI呼び出しの実体

すべてのAI機能は、以下の共通関数を経由して呼ばれます。

```javascript
async function callAI(messages, system){
  const res = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "claude-sonnet-4-6",
      max_tokens: 1000,
      system: system || AI_SYSTEM,
      messages
    })
  });
  if(!res.ok) throw new Error("API " + res.status);
  return parseAIResponse(await res.json());
}
```

- 使用モデルはコード上 `claude-sonnet-4-6` 固定です。
- `max_tokens` は 1000 固定（1回の応答が長くなりすぎないようにする実装上の上限）。
- 通信失敗時（`res.ok`が偽）は例外を投げ、呼び出し元でユーザー向けエラーメッセージに変換されます。

## 3. 3つのAI人格（ペルソナ）

チャット相談機能には、システムプロンプトが異なる3つのペルソナが実装されています。

```javascript
const AI_PERSONAS = {
  announcer: { who: "うさうさアナウンサー🐰", ... },
  director:  { who: "うさうさ監督🎬", ... },
  manager:   { who: "うさうさマネージャー📣", ... }
};
```

### 3-1. うさうさアナウンサー🐰（既定人格）
配信初心者に寄り添う「やさしいベテランアナウンサー」という設定のシステムプロンプトが与えられています。専門用語を避け、実践的なアドバイスを簡潔に返し、最後に一言励ましを添えるよう指示されています。台本本文のみを求められた場合は前置きなしで本文だけを返す、という条件も明記されています。

### 3-2. うさうさ監督🎬（本気レビュー）
「厳しくも愛のあるスパルタ指導」というキャラクター設定です。曖昧な称賛を禁じ、弱点・改善点を具体的に指摘したうえで、必ず改善案とセットで返すよう指示されています。人格ではなく作品の品質のみを評価対象とする旨も明記されています。

### 3-3. うさうさマネージャー📣（応援レビュー）
良い点を具体的に3つ程度挙げて称賛し、伸ばすための小さなコツを1つ添える、という前向き特化のシステムプロンプトです。

どのペルソナも「日本語で答える」ことがシステムプロンプトに明示されています。

## 4. 機能一覧と、それぞれが実際に送るプロンプト

すべて `quickAI()` / `reviewWith()` / `aiFormatMemo()` の各関数で組み立てられ、`callAI()` に渡されます。

| UI上のボタン | 呼び出し関数 | 使用ペルソナ | 送信内容の要旨（コードのテンプレート文言） |
|---|---|---|---|
| ✨ AIで読みやすく（原稿編集） | `quickAI("polish")` | アナウンサー | 「声に出して読みやすいように、自然な区切り・間・やわらかい言い回しで整えて。内容や意味は変えすぎないで。台本本文だけ返して」＋原稿全文 |
| 概要欄用に要約して（クイック相談） | `quickAI("summary")` | アナウンサー | 「YouTubeの概要欄に載せる紹介文（3〜4文）に要約して」＋原稿全文 |
| オープニングあいさつを作って | `quickAI("open")` | アナウンサー | 「明るくやさしい雰囲気のオープニングあいさつを作って」＋（原稿があればテーマ参考として先頭800字） |
| 読み方・間のコツを教えて | `quickAI("read")` | アナウンサー | 「間の取り方・噛まないコツをやさしく教えて」＋（原稿があれば先頭800字を参考として） |
| 🎬 監督レビュー（原稿/メモ） | `reviewWith("director", target)` | 監督 | 「本気で（厳しめに）レビューして。悪い点・直すべき点を具体的に指摘し、改善案も添えて」＋対象全文 |
| 📣 応援レビュー（原稿/メモ） | `reviewWith("manager", target)` | マネージャー | 「良いところを具体的に見つけて全力で褒めて。伸ばすコツを一つ」＋対象全文 |
| ✨ AIでマークダウン整形（メモ） | `aiFormatMemo()` | 整形専用の別システムプロンプト（人格なし） | 「見出し・箇条書き・強調を適切に使ったMarkdownに整形して。内容や意味は変えないで。整形後の本文だけ返して」＋メモ本文 |
| 🐰💬 相談（自由入力チャット） | `sendAI(text, persona)` | 選択中のペルソナ（既定はふわふわしろうさぎアナウンサー※新人さんに優しい、IT研修講師のオリジナルキャラクター） | ユーザーの自由入力そのまま＋直近の会話履歴 |

原稿・メモの本文は `clip()` 関数で送信前に文字数を切り詰めています（既定 4000〜6000文字程度、超過分は「（以下省略）」として切り捨て）。これはAPIへの送信量を抑えるための実装です。

## 5. チャット履歴の扱い

自由入力チャット（🐰💬 相談）では、会話履歴をAPIに渡す前に整形する関数があります。

```javascript
function apiMessages(){
  const src = aiMessages.filter(m => !m._greet).slice(-12);
  const out = [];
  src.forEach(m => {
    const role = m.role === "user" ? "user" : "assistant";
    const content = clip(m.content, 6000);
    if(out.length && out[out.length-1].role === role)
      out[out.length-1].content += "\n\n" + content;
    else out.push({ role, content });
  });
  while(out.length && out[0].role !== "user") out.shift();
  return out;
}
```

事実として確認できる仕様：
- 初回の挨拶メッセージ（`_greet` フラグ付き）はAPIに送らない。
- 直近12件のみを送信対象とする（それより古い会話はAPIには渡らない＝文脈として参照されない）。
- 同じroleのメッセージが連続する場合は1つに結合する（Anthropic APIの「user/assistant交互」制約に合わせるため）。
- 履歴の先頭が`user`でない場合は、`user`になるまで先頭を削除する。

つまり、チャットは「その場のセッション内の直近のやり取り」だけを文脈として保持する設計であり、無制限に過去の会話を記憶し続ける仕組みではありません。

## 6. レビュー結果の保存

AIの応答（挨拶メッセージを除く）には「📚 勉強ノートに保存」ボタンが付与されます。押すと、応答本文がそのまま `state.notes` に `kind: "study"` として追加され、通常のメモと同様に `localStorage` へ保存されます。AIの応答自体をアプリが自動保存することはなく、ユーザーがボタンを押した場合のみ保存されます。

---

## 7. エラー時の挙動

`callAI()` が例外を投げた場合（通信失敗・レスポンス異常など）、アナウンサー人格からの応答として、次のような定型メッセージに差し替えられます（コードそのまま、要約せず引用）。

> 「ごめんなさい🐰 通信でエラーが出ました（〜）。この相談機能は Claude のアーティファクト上で動きます。少し時間をおいて、もう一度お試しくださいね。」

チャット欄はエラーでも入力可能な状態に戻り、再送信をブロックしません。

---

## 8. AI機能が「関わらない」部分

以下は完全にローカル・オフラインのロジックであり、AIは一切関与しません。

- 読み上げの速度計算・スクロール・カウントダウン
- 文字数カウント（`countChars`）とページ分割（`splitPages`）
- ハイライト・しおり・原稿履歴・読み上げ実績ログ
- コントラスト比計算（W3C WCAG 2.1 相対輝度式に基づく独自実装）
- Word(.docx) / Excel(.xlsx) / CSV のインポート（ZIP展開・XML抽出はすべて自前のパーサー）

これらは通常版（ローカル）でも完全に動作します。AIが必要なのは「相談チャット」「読みやすく整形」「レビュー」「Markdown整形」の4系統の機能のみです。

## 9. まとめ：仕組みの一言要約

> このツールのAI機能は、Claudeのアーティファクト実行環境が提供する `fetch` 経由のAPIアクセスを利用して、あらかじめ用意された3種類のシステムプロンプト（アナウンサー／監督／マネージャー）に、ユーザーの原稿・メモ・チャット入力を添えて `claude-sonnet-4-6` に問い合わせているだけです。APIキーの管理・課金・レート制御はすべてアーティファクト実行環境側に委ねられており、本ツール自身はそれらを一切扱っていません。
