# うさうさプロンプター

**発表原稿が、ちょうどいい速さで流れていく。だから、前を向いて話せます。**

勉強会・LT・研修・配信で「原稿を見ると下を向いてしまう」「時間が読めない」——そんな悩みを、ブラウザで開くだけの小さな道具で軽くします。

インストール不要。会員登録もなし。ダウンロードして、ダブルクリックするだけです。

---

## 30秒で試す

1. **[usausa-prompter-v1_7B-PRO.html](tool/usausa-prompter-v1_7B-PRO.html)** を開く
2. 右上の **「Download raw file」** をクリック
3. 保存したファイルを**ダブルクリック**（ブラウザが開きます）
4. 原稿を貼り付けて「再生」

以上です。ネット接続も要りません（AI相談機能をのぞく）。

> 原稿と設定は、お使いの端末の中だけに保存されます。どこかに送信されることはありません。

---

## こんなときに使えます

| 場面 | 効果 |
|---|---|
| 勉強会・LT | 5分の持ち時間を、時間どおりに話しきれます |
| セミナー・研修 | 手元の紙をめくらずに済みます |
| オンライン配信 | カメラ目線のまま原稿を読めます |
| 動画収録 | 撮り直しが減って、収録が早く終わります |

主な機能は、速度調整（80〜600字/分）、文字サイズ変更、縦書き／横書き、ミラー表示（プロンプター機材用）、しおり、タイマー、読み上げ実績の記録、Word / Excel からの原稿取り込みなど。

## このリポジトリの歩き方

目的に合わせて、4つの入口があります。

### すぐ使いたい → [`tool/`](tool/)

ツール本体（HTMLファイル1つ）が入っています。上の「30秒で試す」の手順でどうぞ。
操作に迷ったら [使い方ガイド](docs/04-user-manual.md) をご覧ください。

### 中身を知りたい → [`docs/`](docs/)

「このツール、どうやって作られているんだろう」という方へ。
要件定義から設計、そして全2,204行のコード解説まで、作り方をすべて公開しています。

| ドキュメント | こんな方に |
|---|---|
| [要件定義書](docs/01-requirements.md) | 何を・なぜ作ったのかを知りたい |
| [詳細設計書](docs/02-design.md) | どう作られているかを知りたい |
| [修正・拡張ガイド](docs/03-extend-guide.md) | 自分好みに改造したい |
| [利用手順書](docs/04-user-manual.md) | 使い方を詳しく知りたい |
| [全行コード解説](docs/05-code-walkthrough.md) | コードを頭から読んでみたい |
| [一行ずつ解説](docs/06-furigana-line-by-line.md) | **プログラミングは初めて** |
| [設計書の書き方](docs/07-doc-flow.md) | 実務の文書の流れを知りたい |
| [免責事項](docs/08-disclaimer.md) | 利用条件を確認したい |

初めての方には [一行ずつ解説](docs/06-furigana-line-by-line.md) がおすすめです。
コードに日本語のふりがなを振って、記号の読み方から説明しています。

### 品質を確かめたい → [`tests/`](tests/)

「本当にちゃんと動くの？」を確かめられる、動的テスト20本が入っています。

```bash
cd tests
cp ../tool/usausa-prompter-v1_7B-PRO.html ./tool.html
npm install
npm test
```

テストの書き方を学びたい方は [実践ガイド](tests/GUIDE.md) もどうぞ。

### 自分でも作りたい → [`kits/`](kits/)

**プログラミングの知識は要りません。** 穴埋めするだけで、自作ツールづくりの流れを一通り体験できるキットです。

```
(1) 何に使う？   →  (2) 設計する   →  (3) 手順書を書く  →  (4) 運用する
```

| キット | 内容 |
|---|---|
| [事例集](kits/usecase-kit/) | 受付・集計・チェックなど、職種別の応用アイデア |
| [設計書キット](kits/docs-for-nonit/) | 要件定義・詳細設計・実装仕様の穴埋めシート |
| [手順書キット](kits/manual-kit/) | 使い方ガイドを悩まず書ける型 |
| [返信テンプレ集](kits/support-templates/) | 問い合わせへの返信ひな形 |

記入サンプル付きなので、「何を書けばいいか分からない」で止まりません。

<details>
