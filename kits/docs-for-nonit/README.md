# 非IT職のための「3つの設計書」入門キット

「要件定義書」「詳細設計書」「実装仕様書」――IT の現場でよく聞くけれど、非IT職には正体がつかみにくい3つの文書。このキットは、**読んで理解し、穴を埋めれば自分の1枚が完成する**ことをゴールにした教材です。プログラミングの知識は要りません。

## こんな人向け

- 開発チームと会話する必要が出てきた、企画・営業・事務・管理部門の方
- 「要件定義から」と言われて固まってしまった方
- 外注の見積書や仕様書を読む立場になった方
- 新人研修で「設計書の気持ち」を先につかんでおきたい方

## 3つの設計書は「家づくり」でたとえられます

| 文書 | 家づくりでいうと | 誰が主役か | 答える問い |
|---|---|---|---|
| 要件定義書 | 「どんな暮らしがしたいか」の希望リスト | 施主（依頼する人） | 何を・なぜ作るのか（WHY/WHAT） |
| 詳細設計書 | 設計図・間取り図 | 建築士（作り方を決める人） | どう作るのか（HOW） |
| 実装仕様書 | 現場の施工手順書 | 大工（実際に作る人） | どの部品を・どの順で（具体） |

上から下へ、だんだん具体的になります。この順番が体に入れば、もう迷いません。

## このキットの中身

```
docs-for-nonit/
├─ README.md                         ← いまここ
├─ 00_はじめに_3つの設計書とは.md      ← 全体像。まず読む
├─ 01_要件定義書_穴埋めシート.md       ← 埋める（サンプル記入例つき）
├─ 02_詳細設計書_穴埋めシート.md       ← 埋める
├─ 03_実装仕様書_穴埋めシート.md       ← 埋める
├─ 04_記入サンプル_年賀状アプリ.md     ← 3枚が全部埋まった完成見本
└─ 05_全力サポート_つまずきQ&A.md      ← 手が止まったらここ
```

## 使い方（3ステップ）

1. `00_はじめに` を読んで、3つの文書の役割をつかむ
2. `01`→`02`→`03` の順に、穴埋めシートを上から埋める（分からない欄は空でOK。後で埋まる）
3. 手が止まったら `05_全力サポート` を開く。完成形を見たくなったら `04_記入サンプル`

> うさうさ研修工房の約束：**埋められない欄は「未定」と書いてよい。** 分からないことを分からないと書けるのが、良い設計書の第一歩です。嘘の情報でマスを埋めないでください。
