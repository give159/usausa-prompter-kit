# トラブル対応 返信テンプレ集（非IT職向け）

自作ツールや業務システムについて、お客様・利用者から「動かない」「エラーが出た」と連絡が来たとき、**穴埋めするだけで落ち着いた返信が書ける**テンプレ集です。あわてず、角を立てず、正確に。3ステップ（受領→確認→回答）で対応できます。

## こんな人向け

- 自作ツールを配布して、問い合わせに返信する立場になった人
- 不具合報告への返信文をゼロから考えるのがしんどい人
- 「すみません」ばかりの返信になってしまう人
- 技術は分からないが、窓口対応はしないといけない人

## トラブル対応の基本は「3ステップ」

```
① 受領   … まず「受け取りました」を最速で返す（相手の不安を止める）
② 確認   … 状況を正確に聞き出す（原因の切り分け）
③ 回答   … 解決策、または見通しを伝える（次の一手を示す）
```

多くの人は①を飛ばして②③でいきなり完璧に答えようとして、返信が遅れます。**遅い完璧より、速い受領。** まず「受け取りました」だけでも相手は安心します。

## キットの中身

```
support-templates/
├─ README.md                    ← いまここ
├─ 00_トラブル対応の心構え.md      ← 3ステップの考え方。まず読む
├─ 01_受領テンプレ.md            ← 第一報への返信ひな形
├─ 02_確認テンプレ.md            ← 状況を聞き出すひな形
├─ 03_回答テンプレ.md            ← 解決・見通しを伝えるひな形
├─ 04_状況別テンプレ集.md         ← よくある8ケースの返信例
└─ 05_やってはいけない返信.md      ← NG例と、その直し方
```

## 使い方

1. `00_心構え` で3ステップをつかむ
2. 連絡が来たら `01→02→03` の順にテンプレを開き、`＿＿＿` を埋めて返信
3. よくある状況なら `04_状況別テンプレ集` からコピーして微調整
4. 送る前に `05_やってはいけない返信` で自己チェック

> うさうさ研修工房の約束：**分からないことを、分かるふりで答えない。** 原因が不明なら「調査します」と正直に。憶測で「直りました」と言うのが、いちばん信頼を失います。
