# 04 公開前チェックリスト

手順書を配る前の最終確認です。上から順に確認してください。全部チェックできたら配布OKです。

---

## 中身のチェック

- [ ] ①を読むだけで「これは何のツールで、自分に必要か」が30秒で分かる
- [ ] ②で、始める前に必要なもの（機器・ファイル・ネット有無）が分かる
- [ ] ③の手順を、実際に自分で操作しながら書いた
- [ ] ③は1行1動作になっている（「〜して〜する」が残っていない）
- [ ] ③に「押すとどうなるか（結果）」が書いてある
- [ ] 画面に表示されている言葉と、手順書の言葉が一致している
- [ ] ④に、よくあるつまずきが2つ以上ある
- [ ] ⑤に連絡先がある

## 正確さのチェック（うさうさ研修工房の約束）

- [ ] 想像で書いた手順が1つもない（全部、実際に動かして確認した）
- [ ] 数値（速度・サイズ・上限など）が実物と合っている
- [ ] 「分からないこと」を分かるふりで書いていない
- [ ] 個人情報・パスワード・社外秘を手順書に書いていない

## 読みやすさのチェック

- [ ] 専門用語には、その場で短い説明がある（または使っていない）
- [ ] スマホでも読める長さ・改行になっている
- [ ] 1画面に文字を詰め込みすぎていない

## 最強のテスト（できれば）

- [ ] ツールを知らない人に、手順書だけ渡して操作してもらった
- [ ] その人が詰まった所を、手順書に反映した

## 配布前の事務

- [ ] 更新日とバージョンが最新になっている
- [ ] ファイル名が分かりやすい（例: ツール名_使い方_v1.0）
- [ ] 配布形式を決めた（PDF／印刷／リンク共有など）

---

## 全部チェックできたら

配布してOKです。ツールを更新したら、この手順書も更新日を直して見直してください。手順書は「生もの」です。

> 迷ったときの合言葉:**やって動いた操作だけを書く。**
