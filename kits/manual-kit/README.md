# 悩まず書ける「使い方手順書」作成キット（非IT職向け）

自分で作った（または依頼して作ってもらった）ツールの**使い方手順書**を、穴埋めするだけで時短で完成させるキットです。「何をどの順で書けばいいか分からない」を解消します。文章力は要りません。マスを埋めれば、読む人に伝わる手順書になります。

## こんな人向け

- オリジナルツール・アプリ・Excelマクロ等の「使い方」を配る必要がある人
- 研修や引き継ぎで「操作マニュアル」を作ることになった人
- ゼロから書くと時間がかかって毎回つらい人
- 書いたけれど「読んでも分からない」と言われてしまう人

## なぜ手順書は難しく感じるのか

作った本人は仕組みを知っているので、つい「分かっている前提」で書いてしまいます。でも読む人は初めて。この**知識のズレ**が「分かりにくい手順書」の正体です。このキットは、穴埋めの順番自体が「初めての人の目線」になるよう設計されています。埋めれば自然とズレが埋まります。

## キットの中身

```
manual-kit/
├─ README.md                       ← いまここ
├─ 00_はじめに_伝わる手順書の型.md    ← 手順書の"型"を知る。まず読む
├─ 01_手順書_穴埋めシート.md          ← これを埋める（本体）
├─ 02_記入サンプル_prompterの手順書.md ← 埋まった完成見本
├─ 03_全力サポート_つまずきQ&A.md      ← 手が止まったら
└─ 04_公開前チェックリスト.md         ← 配る前の最終確認
```

## 使い方（最短ルート）

1. `00_はじめに` で手順書の型（5つのブロック）を10分でつかむ
2. `01_穴埋めシート` を上から埋める（分からない欄は `未記入` でOK・後で埋まる）
3. 迷ったら `02_記入サンプル` を隣に開いて見比べる
4. 書けたら `04_公開前チェックリスト` で最終確認して配布

> うさうさ研修工房の約束：**やってみて動いた操作だけを書く。** 想像で手順を書かないこと。1回自分で操作しながら書くのが、結局いちばんの時短です。
